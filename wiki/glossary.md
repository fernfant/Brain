# Glossary

Definitions for all 28 concept pages in this wiki.

---

## A

**[[Andrej Karpathy]]** — AI researcher and educator (ex-OpenAI, ex-Tesla) known for the insight that LLM usage is shifting from code generation to manipulating knowledge stored as markdown files.

## B

**[[Bayut]]** — PF's primary UAE real estate portal competitor; raised a Stake Series B in February 2026 and actively signs new-to-market exclusive inventory.

## C

**[[CLAUDE.md (concept)]]** — A persistent markdown configuration file read automatically by Claude Code each session, encoding project tech stack, coding conventions, workflow rules, and a bug log so context doesn't need to be re-established.

**[[Claude Code]]** — Anthropic's terminal-native agentic coding assistant that reads entire codebases, plans multi-file implementations, executes changes, and self-corrects via validation loops — designed as a development partner rather than an autocomplete tool.

**[[Claude Cowork]]** — Anthropic's desktop application enabling autonomous multi-step task execution with local folder access, serving as the non-technical user's entry point to agentic Claude.

**[[Claude Managed Agents]]** — Anthropic's cloud-hosted agent platform (public beta April 2026) for deploying Claude agents that run 24/7 on Anthropic's infrastructure without requiring server management.

**[[Claude Skills]]** — Reusable workflow packages for Claude Code, consisting of a SKILL.md instruction file plus optional scripts and reference files, allowing multi-step workflows to be encoded once and reused across sessions.

## D

**[[DLD Market]]** — Dubai Land Department transaction data, tracking property sale values and volumes split between Primary (off-plan) and Secondary (resale) markets.

## E

**[[Exclusive Listings]]** — A Property Finder supply metric counting listings available only on PF; rebaselined in February 2026 to remove non-compliant and duplicate entries; ran 30% below its 23K January target throughout Q1.

**[[Exposure LPL]]** — Leads per listing (eLPL), a demand proxy metric visible in client dashboards; Buy eLPL collapsed to 3.7 vs a 7.6 target in March 2026 (–51%) due to the conflict-driven lead decline.

**[[Extended Thinking]]** — Claude's capability to reason step-by-step before responding; implemented as "adaptive thinking" in Claude 4.6 with a configurable effort parameter (high/medium/low) controlling reasoning depth.

## G

**[[Geopolitical Risk]]** — The US-Iran conflict that began in early March 2026, causing a –24% decline in UAE real estate transaction values, a sales forecast crash from 95% to 66%, and a temporary collapse in New Projects demand.

## H

**[[Harness Engineering]]** — The practice of structuring AI interactions through persistent configuration files and automated verification checkpoints to make Claude's behaviour predictable and consistent across sessions, rather than relying on ad-hoc prompting skill.

## K

**[[Kochava]]** — A mobile attribution platform used by Property Finder; experienced tracking failures in W3 (Jan), W10 (Mar), and W11 (Mar) 2026, corrupting app session data and downstream LPL calculations.

## L

**[[LLM Wiki]]** — A persistent, LLM-maintained knowledge base pattern where raw source documents are compiled into structured, interlinked markdown pages; the LLM acts as compiler and librarian, compounding knowledge with every ingested source.

## M

**[[MCP]]** — Model Context Protocol; a standard enabling Claude to connect to and interact with external tools and services, including Obsidian vaults (via mcp-obsidian), GitHub, Figma, Notion, Sentry, and others.

## N

**[[NPS]]** — Net Promoter Score; Property Finder's B2C NPS hit 3 in January 2026 (–9 MoM, –10 YoY), a pre-conflict product health warning during a period of strong usage growth.

**[[New Projects]]** — Property Finder's off-plan/developer property segment; the highest-performing segment in Q1 2026 pre-conflict (+169.7% revenue YoY in January) and the fastest-recovering post-conflict (+29% above target by W13).

## O

**[[Obsidian]]** — A local-first markdown editor used as the wiki IDE; features include backlinks, graph view, Dataview plugin for querying frontmatter, and Marp for slide deck generation.

**[[Obsidian Web Clipper]]** — A browser extension that converts web articles to markdown format for local storage in the raw/ ingestion folder.

## P

**[[Plan Mode]]** — A read-only Claude Code mode (activated via Shift+Tab) where Claude analyses the codebase and develops a full implementation strategy before making any changes, reducing debugging cycles on complex tasks.

**[[Prompt Engineering]]** — The practice of structuring instructions to elicit better, more consistent outputs from LLMs, covering clarity, context provision, examples, XML tagging, chain-of-thought, mode discipline, and prompt chaining.

**[[Property Finder]]** — UAE's leading real estate portal and the subject of this wiki's business data; holds #1 organic search ranking across all UAE Sale keywords and 61% organic traffic share in the Residential Sale segment.

## S

**[[SEMRush]]** — An SEO analytics platform used by Property Finder for keyword ranking data; experienced data reliability issues in W4 January 2026 that made the month-end SEO baseline unreliable.

**[[SEO]]** — Search engine optimisation; Property Finder's most resilient marketing channel through Q1 2026, gaining +3.9pp in the week of the conflict shock and holding at 45.8% through March.

**[[Share of Voice]]** — Property Finder's traffic and ranking share relative to competitors, measured across App sessions (SoS), Web sessions (SoS), SEO (SoV), and PR (SoV) channels.

## U

**[[UAE Real Estate]]** — The Dubai and Abu Dhabi property market; structured around a Primary (off-plan) and Secondary (resale) split, with Sale transactions generating 85% of the total commission pool vs 15% for Rent.

## W

**[[Weekly Business Review]]** — Property Finder's weekly reporting cadence covering commercial performance, demand metrics, product KPIs, and operational health across all business lines.
